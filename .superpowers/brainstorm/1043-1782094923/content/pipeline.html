更新流水线总览

从构建到用户获取更新的完整链路

🏗️ 第一步：构建 & 发布

CI / 手动发布流程
npm run build:win
→
build/Blank Tool Setup 2.0.6.exe
npm run build:mac
→
build/Blank Tool-2.0.6.dmg
关键：electron-builder 发布配置
package.json 中加入 "publish": { "provider": "github", "owner": "purejiang", "repo": "blank_tool" }
然后运行 npm run build:win 时加环境变量 GH_TOKEN=xxx 即可自动上传到 Release

📦 第二步：GitHub Release 存储

GitHub Releases 页面
v2.0.6 (Latest)
Release notes: 修复了 xxx 问题，新增了 yyy 功能
📦 Blank-Tool-Setup-2.0.6.exe 128 MB
🍎 Blank-Tool-2.0.6.dmg 145 MB
🐧 Blank-Tool-2.0.6.AppImage 150 MB

🔄 第三步：应用内检测 & 下载

electron-updater 内部流程
步骤 1
请求 api.github.com/repos/purejiang/blank_tool/releases/latest
步骤 2
解析 JSON，比对 tag_name（如 v2.0.6）vs 当前 app.getVersion()
步骤 3
筛选 assets 中匹配当前平台的安装包（Windows→.exe, macOS→.dmg, Linux→.AppImage）
步骤 4
下载至用户数据目录，校验 SHA512，完成后触发 update-downloaded 事件

🚀 第四步：安装

NSIS 静默安装

下载的 .exe 就是 NSIS 安装包（和你手动安装的是同一个）。
electron-updater 调用 Blank-Tool-Setup-2.0.6.exe /S（静默模式），
安装完成后应用自动重启为新版本。

⚠️ 注意：NSIS 静默安装需要权限（写入 Program Files），如果用户安装路径在系统盘，可能会弹 UAC 对话框。这是 Windows 的安全机制，无法绕过。

🔄 数据流

更新检测 → 下载 → 安装 完整流程
启动
main.ts 中 app.whenReady() 后调用 initAutoUpdater()
注册 autoUpdater 事件监听器，启动后 3 秒自动检查
↓
检查
autoUpdater.checkForUpdates() → GitHub API
仅查正式 Release，跳过 pre-release 和 draft
↓
下载
自动下载到 userData/__update__/
通过 IPC 实时推送进度 (download-progress) 给 renderer
↓
就绪
下载完成，触发 update-downloaded
→ 通知 renderer 显示「更新已就绪，重启后安装」提示
↓
安装
用户点击「重启并安装」→ autoUpdater.quitAndInstall()
NSIS 静默安装 → 应用自动重启为新版本

📡 IPC 通道设计

Renderer → Main (invoke):
check-for-updates → 手动触发检查
quit-and-install → 用户确认重启安装
get-update-status → 获取当前更新状态
Main → Renderer (send):
update-available → 检测到新版本 (版本号 + release notes)
download-progress → 下载进度 (百分比 + 速度)
update-downloaded → 下载完成，等待安装
update-error → 错误信息
update-not-available → 已是最新版本

🎯 关键设计决策

启动时静默检查

  • 延迟 3 秒启动检查，避免影响启动速度
  • 有更新时在后台下载，不打扰用户
  • 下载完成后显示一个小提示条，不弹 modal

手动检查（关于页面）

  • 点击后显示 loading，告知用户正在检查
  • 有更新 → 显示 UpdateDialog（版本号、release notes、下载进度）
  • 无更新 → 显示轻提示「已是最新版本」

跨平台策略

  • Windows: 全自动，NSIS /S 静默安装
  • macOS: 下载后提示用户手动打开 .dmg（Gatekeeper 限制）
  • Linux: 下载 .AppImage，提示用户替换旧文件
